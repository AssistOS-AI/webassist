webAdmin Agent Documentation

The owner-facing agent for lead lifecycle management, lead inspection, and activity reporting.

Overview

webAdmin is an owner-facing management interface for the webAssist data system. It allows site owners to analyze leads, track visitor activity, manage profiling templates, and update site and contact information.

Quick Start

webAdmin is used through the CLI. Run it directly:

node webAdmin/src/index.mjs

You can also send an initial message as a positional argument:

node webAdmin/src/index.mjs "Show me the latest leads"

Exit the interactive session by typing exit, quit, :q, or pressing Ctrl+C.

Webchat Access

webAdmin is also accessible through /webchat?agent=webAdmin.

Data Structure

webAdmin reads data from the shared data/ directory:

FolderPurpose
leads/Lead files created by webAssist ({sessionId}-lead.md)
sessions/Session profile and history files from webAssist
profilesInfo/Visitor profiling templates
info/Website information files
config/Owner contact configuration (owner.md)

Skills and Commands

news

List the most recent leads, sorted newest first. Default shows the latest 5 leads.

Example: "Show me the latest leads"

statistics

Generate performance metrics over time intervals: day, week, month, or year. Reports total sessions, total leads, and leads by profile.

Example: "Show statistics for this month"

lead-info

Display comprehensive details for a specific lead, including profile, contact info, and full session history.

Example: "Show details for the Developer lead"

update-lead

Change a lead's status. Allowed transitions from new: invalid, contacted, or converted.

Example: "Mark the latest lead as contacted"

manage-profile

Create or update visitor profiling templates in profilesInfo/. Each profile defines characteristics, interests, and qualifying criteria used by webAssist to match visitors.

Example: "Create a profile for Enterprise Clients"

manage-site-info

Create or update website information files under data/info/. Use this to update the knowledge base available to webAssist.

Example: "Update the services information file"

manage-owner-info

Update your contact information stored in data/config/owner.md. Supports email, phone, calendar links, and meeting details.

Example: "Update my phone number to +40123456789"

Orchestration Flow

webAdmin uses the admin-flow orchestrator, which processes each owner request by selecting exactly one skill, executing it, and formatting the result in the same language as the request. No manual skill selection is needed—just describe what you need in natural language.

CLI Parameters

ParameterDescription
<message>Owner message text (positional)
--session-id <id>Reuse a specific session id
--data-dir <dir>Override the runtime data directory
--agent-root <dir>Override the agent root for runtime initialization
-h / --helpPrint CLI usage

Testing

Tests are in the tests/ directory with dedicated fixtures. Run:

node tests/runAll.mjs

Or run all test files directly:

node --test "tests/*.test.mjs"
